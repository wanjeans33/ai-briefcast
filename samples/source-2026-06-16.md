### 头条 01：在AI竞赛里追赶的Meta，把你的Facebook公开帖变成了别人搜索时的答案
周一，Meta在Facebook上线了一批新的AI功能，其中最不起眼、却最贴近每个用户的一项叫「AI Mode」。当你在Facebook里搜索时，它会出现在「People」「Marketplace」这些老搜索模式旁边，看上去只是多了一个选项。
但这个选项背后的逻辑和过去不一样。你点进去问一个问题，Meta给你的答案不再只来自官方信息或网页，而是从平台上的公开内容里抓取、再由AI拼成一段回复。换句话说，你随手发的一条公开帖，可能正在变成别人搜索时读到的那段AI答案的来源。
抓取的范围还不止Facebook一家。Meta明确说，AI Mode会跨平台调取它旗下各个产品上的公开信息。你在某个角落发的内容、留下的评论，只要设成公开，就进了这个池子。你当初发帖时面对的是熟人和好友，如今它面对的是一个会被陌生人提问、被AI改写的检索系统——发帖时的语境和此刻被引用的语境，完全是两回事。
Meta如此急着把这些东西铺出来，和它在AI竞赛里的位置有关。它一直被看作追赶的一方，既要拿出能见人的AI产品，又要把用户留在自家平台上多停留一会儿。AI Mode就是这两件事的交叉点:它让Facebook的搜索看起来更聪明，又顺手把用户多年积累的公开内容变成了喂养这套系统的燃料。这一批功能从周一起陆续推送，同时上线的还有能把照片里的球衣一键换成其他队服的预设玩法。

### 头条 02：韩国全民拥抱AI，美国Gen Z采用却停滞，「人人都在用AI做一切」被夸大
「人人都在用AI做一切」是过去一年最流行的判断。来自国家、个人和职业三个维度的信号，正在把它拆开。
先看国家。一名科技记者从旧金山飞抵首尔，刷脸通过无人值守的入境口，一路发现韩国社会对AI的拥抱远超美国，连地铁通勤都被各种AI服务包围。
换到个人，画面立刻分化。一篇热门文章把人们消费AI比作吃肉：有人全盘拥抱，有人刻意节制，有人干脆回避。即便在Gen Z这代人里也一样。过去一年AI采用率几乎停滞，约两成从不使用，所谓「试过就离不开」，在多数人身上没发生。
再看最该被AI改写的软件工程。这个行业监管壁垒少、代码又是AI的强项，但两名研究者梳理证据后判断，没有迹象显示能力越过某个门槛就会带来大规模裁员，工程师至今没被取代。
三组信号拼出同一个结构：真实采用远低于「全民全面使用」的假设。微软新上线的全美AI扩散地图，用聚合数据呈现的正是各地冷热不均。

### 头条 03：号称自研的397B大模型被查出是两个现成模型的加权合并，机构的AI成果正撞上独立核查
里约市政技术机构IplanRIO发布了一个3970亿参数大模型，宣称完全自研。一个名为Nex的AI团队公开比对后发现，把这个模型硬编码的自我介绍去掉，它有79%的概率自称「来自Nex」，自称里约的概率为0，还能一字不差地背出Nex团队编写的组织故事。
更硬的证据在权重本身。这家团队对全部60层逐一比对，发现里约模型的每个权重张量，都是自家模型与Qwen基座的固定混合，配比稳定在0.6比0.4，偏差以数千个标准差计，常规微调做不出这种插值。所谓自研，其实是两个现成模型的加权合并，找不到自己训练的痕迹。
同一周，另一份AI成果也栽在核查上。咨询机构KPMG撤回了一份关于AI使用情况的报告，原因是报告里疑似掺进了AI幻觉。一份教人怎么用AI的报告，自己先成了AI不可靠的证据。

## 二、论文简报（AI 论文简报）
来源页：https://ai-brief.liziran.com/zh/daily/2026-06-16-code-agent-data-sandbox-vla-latent-reasoning.html

### 今日概览
- code agent 的真实瓶颈在「找数据→写代码」的衔接 ：CODA-BENCH 把1009个任务放进平均980个文件的 Linux 沙箱，最强系统成功率只有61.1%，暴露了分开测代码、测数据时看不见的盲区。
- 推理预算可以当资源来分配 ：AVA-VLA 让机器人模型在内部以潜在变量推理，再按置信度提前退出，LIBERO 上98.3%成功率、比显式 CoT 快6倍。
- 扩散训练补上了一整条轨迹的一致性约束 ，这篇 ICML 工作把强化学习里的时序差分搬进去噪过程，作为 drop-in 目标就能提升 FID，步数越少优势越明显。
- 离线又多目标，是很多调参和设计搜索的真实约束 ：DOMOO 用嵌套 Pareto 集学习把多样性做进优化本身，而非事后挑解。

### 重点论文

#### 论文 01：[Agent] 把代码和数据放进同一个沙箱，最强系统也只跑到61.1%
真实开发里，写代码和处理数据从来不是两件事——你得先在一堆文件里翻出哪个数据集有用，再写代码去分析它。CODA-BENCH把这个交织过程搬进了一个基于Kaggle生态的Linux沙箱：每个任务环境平均980个文件，agent得自己在复杂目录里探索、识别相关资源，再生成可执行的分析代码，全套1009个任务覆盖31个领域。结果是当前最强的系统成功率只有61.1%——不是代码写不对,也不是数据处理不了，而是把「数据发现」和「代码执行」这两步串起来时掉链子。这正是分开测试时看不见的盲区:单测代码能力或单测数据能力，分数都不难看，一旦放进真实的文件噪声里就露馅。需要看全文确认它的任务难度分布和评分标准，但这个「合起来测」的思路本身，比又一个排行榜更有信息量。 对从业者来说,更现实的提醒是benchmark类工作的老问题:刷榜容易,迁移到自己的工作流难——61.1%这个数字反映的是Kaggle式结构化数据场景,你自己业务里的脏数据和私有目录结构未必能套用。但如果你在做data agent或autonomous engineer方向,它至少给了一个比纯coding benchmark更接近真实的压力测试。
原始论文标题：CODA-BENCH: Can Code Agents Handle Data-Intensive Tasks?

#### 论文 02：[机器人] 推理预算能不能像资源一样按需分配？
机器人模型用显式思维链（CoT，把推理一步步写成文字）把「看到什么」接到「做什么」，但多步任务里这条链既慢又会逐步放大早期的判断误差。AVA-VLA换了个做法：让推理在内部以潜在变量的形式进行，不再生成文字，再用强化学习给这些潜在轨迹去噪、对齐到任务目标。真正有意思的是「提前退出」——模型按当前状态的置信度自适应决定推到哪一步就收手，等于把推理预算当成可调资源而非固定开销。报告的数字是LIBERO上98.3%成功率、推理比显式CoT快6倍，但更该问的是：省推理在哪些步骤上安全、什么情况下会因为退太早而崩，这需要看全文里的退出策略细节才能判断。
原始论文标题：Think Less, Act Early: Reinforced Latent Reasoning with Early Exit in Vision-Language-Action Models

#### 论文 03：[图像生成] 扩散模型的训练目标，原来漏掉了一整条轨迹
扩散模型生成图像是一步步去噪的，但训练时的目标函数只盯着每一步（或相邻两步）的局部去噪是否准确，从来不检查整条去噪路径前后是否自洽——这个结构性缺口一直被忽视。这篇ICML工作把强化学习里成熟的时序差分（TD，让相邻时刻的预测互相校准）搬过来，把去噪过程重新表述成马尔可夫奖励过程，去噪就成了RL里的策略评估问题。没想到这套跨领域借用真的奏效：作为drop-in目标加进去就能提升FID，而且采样步数越少、优势越明显，正好补在「少步快速采样质量会掉」这个实际痛点上。它对离散和连续两种扩散形式都通用，但具体能换来多少质量提升，得看你的步数预算和基线模型，论文只在标准设置下验证，落到自己场景还要实测。
原始论文标题：Temporal Difference Learning for Diffusion Models

#### 论文 04：[训练优化] 当目标函数贵到评估不起，只剩一份离线数据怎么求多目标最优？
调参、资源分配、设计空间搜索这类问题往往有两个现实约束叠在一起：要同时权衡多个目标，而且每次评估目标函数要么拿不到、要么贵到跑不起，只能靠一份固定的离线数据集。这种「离线多目标优化」的麻烦在于代理模型对没见过的设计不准（OOD问题），优化器容易被误导到Pareto前沿之外、还偏向极端值。DOMOO的思路是把「多样性」直接做进流程：用一个累积风险控制模块约束生成解别跑出数据分布太远，再用嵌套的Pareto集学习（PSL）同时学偏好和参数，去适应不同形状的Pareto前沿。它还专门设计了一个离线版的IGD指标来做最终筛选，避开了常用的hypervolume指标偏向极端解的毛病。论文称在合成与真实benchmark上收敛性和多样性的平均排名都最好，不过这是个ICML接收的方法工作，实际表现还要看你自己的数据集和目标维度。
原始论文标题：Diversity-Driven Offline Multi-Objective Optimization via Nested Pareto Set Learning
